What's new in

  • Initial M24 support
    • Adding M24 game directory setting and initial schema.
v4.5.3
  • Schedule Editor - Week 18 quick fix
    • Adding week 18 to the schedule editor if the game save is Madden 22 or 23.
v4.5.2
  • Import bug fixes
    • Fixed an issue when importing a table with auto-save enabled.
    • Fixed an issue when importing a table without every column defined.
v4.5.1
  • Full M23 support
    • Fixed navigations for Madden 23 saves.
  • Bug fixes
    • Fixed a bug where the app would show an infinite spinner if table editor was selected to open by default.
    • Fixed a bug where importing certain tables caused an infinite loading spinner due to invalid enum values.
v4.5.0
  • Tabs!
    • Added the ability to open multiple tables at the same time in different tabs.
    • Open references in a new tab by clicking the middle mouse button on the reference link (or right click on the cell -> Open in new tab).
    • Updated the app layout to make a tab for each screen.
  • Export Error
    • Fixed an issue while exporting a table. An error message will no longer pop up if you choose to auto-open the file in Excel.
v4.4.1
  • Trailing zeros
    • Fixed an issue where editing the very last table in a file caused a game crash when that table had strings.
  • Iconic
    • Updated the app's icon (finally).
  • Updates
    • Updated Electron and handsontable versions. Updated the table editor font and size.
v4.4.0
  • Nice reference!
    • Added the ability to see a list of references to a record. Right click on the record -> 'Find references to this record...' to see the list. You can click on any of these items to open the table.
v4.3.6
  • FTC strings fix
    • Fixing an issue when changing a string in an FTC file. Now correctly updating the table header to account for the new table length.
  • Raw table import
    • There is a new option to import raw data into a table. This is an advanced feature, use it carefully because it can easily crash the game. Find this option under Tools -> Raw table import.
v4.3.5
  • Data import fix
    • When importing a table, the app will automatically recalculate the new empty record references and apply them.
v4.3.4
  • Text field fix
    • Fixed an issue where tables with multiple text fields saved incorrectly when the fields changed.
v4.3.3
  • Array Table Fix
    • Before this update, any change to array table sizes caused the game to crash. This has been addressed. (Any table ending in [])
  • Empty Records
    • Madden franchise files have a complicated algorithm for tracking empty records. The app will now track these empty records to prevent the game crashing.
    • You can manually empty a row by clicking on the row and selecting 'Set selected record(s) as empty'
    • [Advanced] Each table's header points to the first empty record in a table. You can manually set this record by right clicking -> Advanced -> Set as next empty record to use. This can very easily cause the game to crash if you don't know what you're doing. You've been warned!
  • Excel After Export
    • Added an option to toggle whether Excel (or default spreadsheet program) opens automatically after a table export has been completed. Previously, there was no way to disable this behavior. It can become annoying when you want to export a lot of tables in a row.
  • Export Table and FRT
    • Added an option under the Tools menu to export a table to a file that can be opened in a hex editor. Helpful for debugging.
    • Added a similar option to export the entire decompressed FRT file. This will contain every table in the file. Also helpful for debugging.
